# Submission Checklist

Use this checklist before final submission.

## Repository requirements

- [ ] Repository is public and accessible.
- [ ] Root `README.md` includes setup, run, architecture, and testing info.
- [ ] `testbench/` folder exists with grader setup and test instructions.
- [ ] `Evaluation.md` exists with reproducible evidence summary.
- [ ] No secrets committed (`.env` excluded).

## Functional readiness

- [ ] Backend starts and `/health` returns success.
- [ ] Frontend starts and can reach backend.
- [ ] Auth, confidence mode selection, and workspace flow work.
- [ ] Governance gating is visible (`ALLOWED` / `NEEDS_APPROVAL` / `BLOCKED`).
- [ ] Ledger endpoint returns recent events.

## Deliverables

- [ ] GitHub repository link ready.
- [ ] Project video link ready (1-2 minutes).
- [ ] PDF documentation prepared and exported.
- [ ] Citations for external tools/APIs/libraries included in documentation.
